# ADR-008 Big-Bang 컷오버 (MySQL → PostgreSQL)

**상태**: Accepted (2026-05-11)
**관련 카드**: DBM-12 (컷오버 룬북 + 실행)
**입력**: `tasks/db-migration-audit.md` §13.4 (전략) + §13.5 (다운타임 윈도우)

## 결정

MySQL → PostgreSQL 컷오버는 **big-bang 점검 모드 30~60 분**.

- **시간대**: 새벽 04:00~05:00 KST (예비 06:00 까지). 영업 외 시간, 매장 영향 최소.
- **사전 공지**: T-72 h / T-24 h / T-1 h 3 회. 매장에 공지 (sms / line / 관리자 공지 모듈).
- **흐름** (단계별 룬북은 DBM-12 산출): 점검 모드 진입 → 주문 차단 → MySQL 마지막 dump → pgloader 실행 → 정합성 검증 → `DATABASE_URL` 교체 → 서버 재기동 → smoke test → 점검 해제 → T+24 h 모니터링.
- **롤백 윈도우**: 컷오버 후 24 h.

## 이유

### 베타 단계의 트래픽 현실

- 새벽 시간대 주문량 ~0. 다운타임 손실이 사실상 0.
- 식당 / 손님 모두 점검 공지 수용도 높음 (신규 시스템 이전이라는 인식).
- 단일 매장 / 단일 VM 운영 — 듀얼라이트 / read replica 도입의 운영 부담이 이득보다 큼.

### 운영 단순성

- big-bang 은 룬북이 단일 일정표로 표현됨. 단계별 명령 + 검증 + 담당이 명확.
- 사고 발생 시 롤백 경로가 명확 (DATABASE_URL 원복 + 신규 행 역복사).
- ADR-007 의 pgloader 와 자연스럽게 결합 (스테이징 리허설 그대로 운영 실행).

### 롤백 비용 ≤ 듀얼라이트 운영 비용

- 듀얼라이트 도입의 비용: 모든 write 라우터에 분기 + 정합성 검증 / 비상 차단 코드 + 운영 모니터링. **수 카드 분량**.
- big-bang 의 롤백 비용: 24 h 윈도우 내 역복사 스크립트 (`tools/rollback_resync.py`) **하나**. 사전 준비 가능.

## 대안

- **듀얼라이트 (양 DB 동시 쓰기 + 정합성 비교 → 컷)**: 코드 분기 / 정합성 모니터링 / 롤백 모두 복잡. 다운타임 0 이 필요한 규모 (식당 수백 +, 24/7 트래픽) 가 도래하면 재검토. 본 시점 ROI 매우 낮음.
- **Read replica 기반 컷오버 (GCP DMS)**: ADR-007 pgloader 선택과 도구 모순. DMS 자체 비용 + 설정 부담. 본 시점 불필요.
- **Live migration / zero-downtime**: 단일 GCP VM + 단일 DB 운영에서는 환상. 인프라 다중화부터 선결.
- **점검 시간 단축 (예: 5~10 분)**: pgloader + 정합성 검증의 실측 시간이 데이터 규모상 가능하지만, 마진을 두지 않으면 사고 시 즉시 롤백 압박. 30~60 분 마진을 두는 것이 운영자 단독 (자이라) 의사결정 부담 완화에 유리.

## 결론

DBM-12 카드의 두 단계:
1. **룬북 작성 (Phase F-1)**: `tasks/db-migration-runbook.md` 신규. 시간표(T-30 ~ T+30) + 단계별 명령 + 담당 + 검증 + 롤백 절차.
2. **실행 (Phase F-2)**: 스테이징 리허설 (룬북 그대로) → 운영 컷오버. data-migration-engineer (sonnet) + 운영자(자이라) 매 단계 승인.

**롤백 트리거 조건** (룬북에 상세):
- smoke test 실패
- 에러율 > 5 %
- 결제 실패율 > 1 %
- WS 연결 실패율 > 10 %

**미래 분기점**: 식당 수백 + 24/7 트래픽 + 다운타임 1 분 미만 요구 시 듀얼라이트 / DMS read replica 컷오버로 재검토. 본 ADR 을 superseded 처리.

**운영자 협의 항목**: OPR-11 (컷오버 날짜·시간 확정, 사전 공지 채널), OPR-12 (룬북 T-5 단계의 `.env` `DATABASE_URL` 교체 권한).
