## 1. 一行で言うと

## 3. ビフォー / アフター 比較

### ある日の居酒屋・夜営業

| 場面 | 導入前 | 導入後 |
|---|---|---|
| お客様の到着 | 「すみません、メニューください」と呼ぶ → 待つ | テーブル着席 → QRをスキャン → 即メニュー表示 |
| 注文を取る | スタッフが伝票を持ってテーブル巡回 | お客様が自分で注文。スタッフは料理運搬に専念 |
| 厨房への伝達 | 紙の伝票 / 口頭 / 聞き返し | リアルタイムでタブレットに表示。聞き間違いゼロ |
| 追加注文 | 再びスタッフを呼ぶ → 待たせる | スマホからいつでも追加可能 |
| 外国人客 | 指差しメニュー / 翻訳アプリで苦労 | 言語切替ボタンで瞬時に英語/中国語/韓国語 |
| 会計 | レジで明細を計算・確認 | テーブル別合計が自動計算済み。1タップで完了 |
| 閉店後 | 紙伝票を集計、Excelに転記 | ダッシュボードに本日の売上が自動集計 |

## 4. お客様の体験 — 5ステップ

1. **テーブル着席** — 友人と席に着く
2. **QRをスキャン** — スマホのカメラを向けるだけ。アプリ不要
3. **メニューを選ぶ** — 写真付きで選びやすい。言語も切替可能
4. **注文ボタンを押す** — 厨房に瞬時に届く
5. **料理を待つ・楽しむ** — 追加注文もスマホから

> 平均所要時間：注文確定まで **2分以内**

## 6. 解決する 4 つの飲食店の課題

### 課題 1 ─ 人手不足
**Before:** 店長と1人のアルバイトで20席を回す → 注文取りに追われ、料理の質低下
**After:** 注文受付がゼロになり、スタッフ1人分の余裕が生まれる

### 課題 2 ─ 伝達ミス
**Before:** 「カラアゲ2つ」が「3つ」になる → クレーム → 再調理ロス
**After:** デジタルで厨房タブレットに直送 → ミス率ゼロ

### 課題 3 ─ インバウンド対応
**Before:** 英語ができるスタッフがいないと外国人客を断る
**After:** 4言語自動翻訳 → 観光客の口コミ・リピート増加

### 課題 4 ─ 経営判断の遅れ
**Before:** 紙伝票を月末に集計 → 一ヶ月遅れの判断
**After:** リアルタイム売上 → 翌日のメニュー戦略に反映

## 9. 一歩進んだ機能

### 食べ放題・飲み放題
- テーブル単位でセッション開始 → 90分カウントダウン
- 対象メニューに「🍴 食べ放題対象」バッジが自動表示
- ラストオーダー時刻に近づくと黄色警告
- 会計時、コース合計を自動計算

### ランチ／ディナー切替
- 時間と曜日を指定するだけで、メニューが自動的に切り替わる
- スタッフによる毎日の切替作業が不要

### マルチ言語自動翻訳
- 日本語で1回登録すれば、Gemini AI が英語・韓国語・中国語に自動翻訳
- 翻訳結果は管理画面から手動修正可能

### スタッフ管理
- マスターPIN ＋ 個別スタッフ PIN で権限管理
- 「キッチンのみ」「レジのみ」など役割別アクセス制御

## 13. インフォグラフィック化のヒント（NotebookLM 向け）

NotebookLM でビジュアル化する際は、以下の構造を推奨します。

### 推奨レイアウト案

**A. 縦スクロール型（SNS共有用）**
1. ヒーロー：「QRakuで、注文受付時間を 70% 削減」
2. ビフォー/アフター対比（4課題）
3. 5ステップのお客様体験
4. 数字（2,400店舗・98%・60日間）
5. 料金プラン
6. CTA：60日無料トライアル

**B. 1枚A4横向き（営業資料用）**
- 左1/3：課題と解決
- 中央1/3：機能一覧（アイコン付き）
- 右1/3：数字・料金・連絡先

**C. ストーリーボード型（プレゼン用）**
- 1日の流れ（朝→閉店）に沿って機能を配置
- 各時間帯で「QRakuがやっていること」を解説

### 強調すべき差別化ポイント

1. **アプリ不要** — お客様にダウンロードを求めない（重要！）
2. **3分で開始可能** — 他システムは数日かかる
3. **8テーマで個性表現** — お店のブランド感を残せる
4. **食べ放題まで対応** — 他にはない機能
5. **60日間無料** — 業界最長クラス
6. **データ公開で割引** — オーナー優位の珍しい料金体系

### 避けたい表現
