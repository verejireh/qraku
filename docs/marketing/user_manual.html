はじめに —
60秒でQRakuがわかる

QRakuは、お客様の手元のスマートフォンを、注文端末に変えるサービスです。テーブルにQRコードを置くだけで、お客様は専用アプリを入れることなく、メニュー閲覧から注文・決済までを完了できます。

QRakuがやること

  • お客様がQRコードをスキャンすると、お店専用の注文ページが開きます。
  • お客様は写真付きメニューから選び、カートに入れて、注文ボタンを押します。
  • その瞬間、厨房のタブレットに注文が表示されます。紙伝票も、口頭での伝達も不要です。
  • 料理を提供したらステータスを「完了」に。お客様はレジで会計するか、テーブルで決済します。
  • その日の売上は、ダッシュボードに自動で集計されています。

あなたが管理する3つの画面

画面使う人役割
管理画面 (Admin)オーナーメニュー・テーブル・決済設定・分析。お店の頭脳。
キッチン画面厨房スタッフ注文をリアルタイムで受信し、調理状況を更新。
レジ画面レジ担当テーブル別の合計を確認し、お会計を完了する。

本ガイドは、この3つの画面を、それぞれの役割の方が迷わず使えるよう、順を追って説明します。

💡 推奨デバイス キッチン画面は10インチ以上のタブレットがお勧めです（油・水に強いケースを併用してください）。レジ画面はスマートフォンでも操作可能ですが、長時間使うならタブレットが快適です。

最初の30分 —
開店までのチェックリスト

QRakuは、最短3分でアカウント作成、追加で約30分でお店全体の運用を開始できます。以下のチェックリストに沿って進めてください。

アカウント作成（5分）
qraku.com にアクセス → 無料で始める → メールアドレス・店舗名・カテゴリを入力。60日間の無料トライアルが即座に始まります。
テーマを選ぶ（2分）
管理画面の「基本情報」から、お店の雰囲気に合う8つのテーマ（桜・椿・紫陽花・コスモス・ひまわり・ラベンダー・竹・茶花）から1つを選びます。お客様が見るメニュー画面のデザインがこれで決まります。
メニューを5品登録（10分）
「メニュー管理」から看板メニューを5品登録。写真は1枚ずつアップロード、価格と説明文を入れます。これだけで運用は始められます。残りは後で。
テーブルを作成（5分）
「テーブル管理」から店内のテーブル数だけ追加。1番、2番…と通し番号で大丈夫です。
QRコードを印刷して貼る（5分）
テーブル全QRコード印刷 ボタンでA4横向きPDFが出力されます。各テーブルの番号付きQRをカット → ラミネート → テーブルに貼付。
決済方法を決める（3分）
「決済設定」から3つから選択。最初は「カウンター会計」のみで運用を始め、後からSquareやPayPay Directを追加することもできます。
⏱ 完了の目安 ここまでで約30分。あとは厨房タブレットでキッチン画面を開いておけば、お客様の注文を受ける準備は完了です。

メニューを登録する

QRakuのメニュー管理は、写真・価格・説明・カテゴリ・オプション・在庫切れまで、すべて管理画面から数秒で更新できます。スマホからも操作可能です。

基本登録の流れ

「メニュー管理」を開く
管理画面の左メニューから「メニュー管理」をクリック。すでにあるメニューが一覧で並んでいます。
+ 新規メニュー を押す
右上の追加ボタンから新規登録画面へ。
写真をアップロード
スマホで撮った料理写真をそのままアップロード可能（推奨：明るい自然光、皿全体が入る正方形）。Google Cloud Storageで自動最適化されます。
商品名・価格・説明文を入力
日本語で入力すれば、英語・韓国語・中国語にGemini AIが自動翻訳します。手動修正も可能。説明文には素材のこだわりや調理法を入れると注文率が上がります。
カテゴリを選ぶ
「前菜」「メイン」「ドリンク」など、お客様が探しやすい分類を。カテゴリ自体も「カテゴリ管理」から自由に追加できます。
オプションを設定（任意）
「サイズ：S/M/L」「辛さ：普通/中辛/大辛」「トッピング：チーズ +¥100」など。価格加算もここで設定。お客様は注文画面でチェックするだけ。
保存 を押す
即座にお客様の注文画面に反映されます。再起動も再読み込みも不要です。

よく使う操作

やりたいこと操作
本日の品切れにするメニュー一覧の 在庫切れ トグルをON。お客様画面で「Sold Out」表示に。
表示順を変えるドラッグ&ドロップで並び替え。お勧めは上に。
季節限定で一時的に隠す非表示 トグル。削除せず残せます。
価格をまとめて調整一覧で価格欄を直接編集 → タブキーで次へ。Excelのように。
💡 写真のコツ 料理写真は注文率に直結します。スマホで十分ですが、窓際の自然光・上から30度の角度・白い皿。この3つを意識するだけで、プロ並みの一枚が撮れます。

テーブルを作り、
QRコードを印刷する

QRakuには2種類のQRコードがあります。テーブルごとに発行する「イートインQR」と、テイクアウト用の「テイクアウトQR」です。

イートインQRの設置

「テーブル管理」を開く
管理画面 → テーブル管理。現在登録されているテーブル一覧が表示されます。
テーブルを追加する
+ テーブル追加 をクリック → テーブル名・座席数を入力。座席数は後で売上分析に使います。
QRコードをまとめて印刷
右上の テーブル全QRコード印刷 をクリック。A4横向きPDFが生成され、すべてのテーブル分のQRが番号付きで配置されます。
テーマに合わせたデザインで印刷
「QRビルダー」を使うと、お店のテーマに合わせた装飾入りQRも作成可能。桜の花柄・木目調など、メニューと統一感のあるデザインで。
ラミネートして貼付
水濡れ・油汚れ対策に必ずラミネートを。100均でも入手可能。アクリルスタンドに入れると見栄えも◎。

テイクアウトQRの設置

テイクアウト用は1枚だけです。レジカウンター・店頭ガラス・SNSなどに貼って、お客様が事前注文できるようにします。

  • 店頭設置： 営業時間外でも翌営業日の予約注文を受けられます。
  • SNS掲載： Instagramのプロフィールにリンクとして掲載すると、見た瞬間に注文できます。
  • 名刺サイズ： お客様にお持ち帰りいただける名刺型のQRも好評です。
⚠ 注意 テーブルQRには番号情報が埋め込まれています。間違ったテーブルに貼ると、注文がそのテーブルに紐づいてしまいます。印刷後は必ず番号を確認してから設置してください。

決済を設定する —
3つの選択肢

QRakuは、お店の状況に合わせて3種類の決済方式を選べます。あとから変更も追加もできるので、最初は一番シンプルなものから始めましょう。

3つの決済方式

方式初期費用使い勝手こんなお店に
カウンター会計0円注文だけ受けるすぐに始めたい / 既存レジを使い続けたい
Square 連携0円（Squareアカウント要）カード・PayPay・電子マネーSquareをすでに使っている / POS連携したい
PayPay Direct0円（PayPay加盟店契約要）PayPayのみ・低手数料PayPay利用客が多い / 手数料を抑えたい

A. カウンター会計のまま使う

何も設定しなければ、これが既定です。お客様は注文だけスマホで完了し、お会計は従来通りレジカウンターで行います。導入初日からトラブルなく始められる方法です。

B. Square 連携を有効にする

管理画面の「決済設定」を開く
3つのトラックから「Square 連携」を選択。
Squareに接続 をクリック
Square OAuth画面が開きます。Squareアカウントでログインし、「QRakuにアクセスを許可」を選択。
ロケーションを選択
複数店舗を運用している場合は、対象店舗を選択。これでQRakuの売上がそのSquareロケーションに記録されます。
動作モードを選ぶ
EatIn（イートイン）： 注文がSquareのプリンターやKDSに転送されます。
TakeOut（テイクアウト）： お客様がカード情報を入力し、その場で事前決済。Square Web Paymentsで安全に処理されます。

C. PayPay Direct を有効にする

SquareやAirレジを介さず、QRakuがPayPayと直接連携します。手数料が抑えられ、入金サイクルも短くできます。

PayPay for Business に加盟店契約
PayPay公式サイトから加盟店申請。審査後、Merchant ID・APIキーが発行されます。
QRakuに認証情報を登録
「決済設定」→ PayPay Direct → Merchant ID / API Key / API Secret を貼り付け → 保存。
テスト注文で動作確認
サンドボックスモードでテスト注文 → PayPay画面で承認 → 注文完了画面まで遷移すれば成功。
💡 組み合わせ運用 「テイクアウトはPayPay Direct、イートインはカウンター会計」のような組み合わせも可能です。お客様の動線に合わせて柔軟に設計してください。

お客様の体験を、
自分で確かめる

導入直後、必ず一度ご自身で「お客様役」になって注文してみてください。これだけで、お客様がつまずきそうなポイントが見えてきます。

5分の体験テスト

スマホでテーブルQRをスキャン
普段使っているスマホで、印刷したQRコードをカメラで読み取ります。専用アプリは不要です。
メニュー画面のデザインを確認
写真は鮮明か、価格は読みやすいか、説明文に誤字はないか。お客様目線で確認します。
3〜4品をカートに入れて注文
数量変更・オプション選択・カート確認の動線をなぞる。違和感があれば、すぐにメニュー側で改善できます。
キッチン画面で受信を確認
注文ボタンを押した瞬間、厨房のタブレットに注文カードが現れることを確認します。これがQRakuの心臓部です。
言語切替を試す
画面右上の言語切替で英語・韓国語・中国語に切り替えると、メニュー名・説明文が翻訳されます。インバウンド対応の確認に。

キッチン画面の使い方

キッチン画面はQRakuの中で最もリアルタイム性が高い画面です。WebSocketで注文が即座に届き、調理状況を更新するとお客様にも反映されます。

画面の構成

エリア役割
注文待ち新規受信した注文。まだ調理に入っていないもの。
調理中「受付」を押して調理開始した注文。
調理完了「完了」にして提供待ち、または提供済みの注文。

注文を処理する流れ

新着注文音で気づく
注文が入るとアラート音が鳴り、注文待ちエリアに新しいカードが表示されます。テーブル番号・注文時刻・品目・オプション・備考が一目でわかります。
受付 を押して調理開始
カードが「調理中」エリアに移動。お客様の注文履歴ページにも「調理中」と表示されます。
料理ができたら 完了
提供準備が整ったら「完了」を押す。スタッフはこの表示を見て料理を運びます。
必要に応じて品目別に細かく管理
1注文の中で「先にドリンクだけ提供」など、品目単位でステータス変更も可能。
🍳 厨房オペレーションのコツ キッチン画面は「注文待ち」を空にすることを目標に動かすと、自然と回転率が上がります。「調理中」のカードを古い順に並べ替えれば、提供漏れが防げます。

運用モードの切替（KDS と Square 転送）

管理画面の「基本情報 → 注文確認方式」で、以下の2つから選択できます。

  • KDSモード（既定）： QRakuのキッチン画面で注文を受ける。タブレット1台で完結。
  • Square転送モード： 注文が自動でSquareのキッチンプリンター/KDSへ流れます。Squareをすでに運用している店舗向け。

レジ画面 —
お会計の流れ

レジ画面は、テーブル別のお会計を素早く完了させるためのシンプルな画面です。注文した品目・合計金額・食べ放題コースなどがすべて自動計算されています。

テーブル別会計

「会計待ち」一覧から選ぶ
レジ画面を開くと、「会計待ち」エリアにすべてのテーブルが並びます。お会計をご希望のテーブル番号をタップ。
注文明細を確認
注文した品目・数量・単価・小計が表示されます。食べ放題セッションがあれば、コース料金も自動加算。
割引やクーポンを適用（任意）
サービス品引き、サービス料、ポイント割引などを必要に応じて加える。
決済方法を選んで 会計完了
現金 / カード / PayPay / その他 から選択。Square連携時はカード端末と連動。
テーブルがリセットされる
会計完了するとそのテーブルは新規受け入れ可能状態に戻ります。次のお客様をご案内できます。

事前決済（テイクアウト）の場合

Squareテイクアウトモード・PayPay Directを設定している場合、お客様はQRから注文した時点ですでに支払いが完了しています。レジでは「お渡し完了」のチェックのみで作業終了です。

スタッフ画面 —
テーブル状況の把握

スタッフ画面は、店内全体の状況を俯瞰で見るためのフロアスタッフ向け画面です。「今どのテーブルが何をしているか」が一目でわかります。

テーブルカードの色で状況を判断

色状態スタッフのアクション
グレー空席新規お客様をご案内
青注文待ちQR説明や水のサーブ
赤（点滅）調理中・新着注文料理が出たら配膳
黄会計待ちレジへご案内、または席で精算
緑食べ放題進行中残り時間・ラストオーダーを把握

テーブルカードでできること

  • テーブル詳細を開く： 注文履歴・お客様人数・滞在時間を確認。
  • 呼び出し受信： お客様が「店員を呼ぶ」を押した場合、ここに通知が届きます。
  • テーブル移動： お客様が席を移動した場合、注文情報を別テーブルへ移すことができます。

食べ放題・ランチタイムの運用

QRakuは「メニューグループ」という独自の仕組みで、時間帯メニュー切替（ランチ／ディナー）と食べ放題コースを統一的に管理できます。

3種類のメニューグループ

種類動作使い道
時間帯（TIME_WINDOW）指定の時間・曜日にだけメニューが表示されるランチ限定セット / 平日限定 / 朝定食
コース（COURSE）テーブルでセッション開始 → 90分間 食べ放題食べ放題 / 飲み放題 / 90分食べ飲み放題
手動（MANUAL）ON/OFFスイッチで自由に表示切替本日のおすすめ / 売り切れ早期終了

食べ放題コースを運用する

コースを作成
「メニュー管理」→ メニューグループ → + グループ作成 → COURSE を選択。1人あたり料金・時間（分）・ラストオーダー前タイミング・対象メニューを設定。
スタッフ画面でセッション開始
お客様が席に着いたら、スタッフ画面の「食べ放題」タブから対象テーブルをタップ → コース選択 → 人数入力 → 開始。
お客様には残り時間バナー
お客様の注文画面の上部にカウントダウンバナーが表示。ラストオーダー時刻が近づくと黄色に変化し、自動で「ラストオーダー」を促します。
対象メニューは自動的に¥0
食べ放題対象メニューには「🍴 食べ放題対象」バッジが表示され、注文されてもお会計に加算されません。コース料金 × 人数のみ計上。
レジで自動精算
会計時、コース合計が自動表示されます。スタッフが暗算する必要は一切ありません。

売上を読む —
ダッシュボードと分析

毎晩の閉店後、5分だけダッシュボードを見る習慣をつけてください。お店を伸ばすヒントが、データの中に必ずあります。

管理画面のダッシュボード（毎日見る）

  • 本日の売上： リアルタイムで更新。目標達成度の把握に。
  • 注文数： 客数の目安。テーブル数で割れば回転率の概算が出ます。
  • カテゴリ別売上： ドリンク・前菜・メインなどの構成比。
  • 時間帯別客数： ピーク時間とアイドルタイムが視覚化されます。
  • 人気メニュー上位： 注文数ランキング。新メニューの参考に。

分析画面（週1で見る）

  • 曜日別売上： 週末ピーク・平日ボトムの差を把握。
  • 新規/リピーター比率： ポイントシステムONの場合に表示。
  • メニュー別収益貢献： 人気だけでなく利益貢献度を可視化。
  • テーブル別売上： 特に売れる席・売れにくい席が判明。
💡 データの読み方 「売上が下がった日」より「予想より売上が高かった日」に注目してください。何が違ったのか（天気、イベント、新メニュー）が次の打ち手のヒントになります。

よくあるご質問

Q. お客様がスマホを持っていない場合は？

スタッフ用タブレットでお客様の代わりに注文を入力できます。「スタッフ画面」→ 対象テーブル → 注文ボタンから。従来のオーダー方式と並行運用できます。

Q. Wi-Fiが切れたら？

注文受付には客側のモバイル回線が使われるため、お店のWi-Fiが切れてもQRakuは動きます。ただし、店舗側のキッチン画面・レジ画面はインターネット接続が必要です。

Q. 既存POSとの併用は？

可能です。Square・スマレジ・Airレジとの連携も提供しています（Square連携が動作確認済、他は実装中）。詳しくはサポートまでお問い合わせください。

Q. メニューを大量に登録する方法は？

CSVアップロード機能で一括登録が可能です。Excelで作成 → CSV保存 → アップロードするだけ。100品でも10分で完了します。

Q. 解約はいつでもできますか？

はい。管理画面の「サブスクリプション」から1クリックで解約できます。違約金や残期間の請求は一切ありません。データはお客様のリクエストでエクスポート提供します。

Q. 店舗を増やしたら、別契約が必要？
