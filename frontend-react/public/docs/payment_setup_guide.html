2. Square 決済の導入手順
OAuth連携で簡単に接続 — 所要時間：約10分

1Square アカウントを作成する

まず、Squareの公式サイトでビジネスアカウントを作成してください。

link https://squareup.com/jp/ja
  • メールアドレスとパスワードを登録
  • ビジネス情報（店舗名、業種、住所）を入力
  • 銀行口座情報を登録（売上の入金先）
  • 本人確認書類のアップロード（必要に応じて）
lightbulb
既にSquareをご利用中の店舗様へ：
手順1と2はスキップし、そのまま「手順3：QRaku 管理画面で Square を連携する」へ進んでください。

2Square Developerダッシュボードで設定確認

QRakuとSquareを連携するために、Square Developerダッシュボードでアプリケーション設定を確認します。

link https://developer.squareup.com/apps

以下の権限 (Scopes) がQRaku側で自動的に要求されます：

  • PAYMENTS_WRITE — 決済の作成
  • PAYMENTS_READ — 決済情報の読み取り
  • ORDERS_WRITE / ORDERS_READ — 注文管理
  • MERCHANT_PROFILE_READ — 加盟店情報の読み取り
info
店舗オーナー様がDeveloperダッシュボードで操作する必要はありません。QRakuの管理画面から「連携する」ボタンを押すだけで、OAuth認証が自動的に行われます。

3QRaku 管理画面で Square を連携する

QRakuの決済設定ページで以下の操作を行います：

  1. 管理画面 → 「決済設定」 ページを開く
  2. 決済方式から 「Square 決済」 を選択
  3. 「Squareアカウントを連携する (OAuth)」 ボタンをクリック
  4. Squareのログイン画面が表示されるので、ログインして「許可する」をクリック
  5. 自動的にQRaku管理画面に戻り、「連携済み」と表示されれば完了 ✅
lightbulb
連携後、Squareダッシュボードでロケーション（店舗）が正しく選択されていることを確認してください。複数店舗がある場合、デフォルトのロケーションが使用されます。

4Square Reader / Terminal の準備（対面決済の場合）

店内のテーブル決済でクレジットカードを受け付ける場合は、Square端末が必要です：

端末 価格 特徴
Square Reader ¥7,980 Bluetooth接続。スマホ・タブレットと連携。タッチ決済・ICチップ対応
Square Terminal ¥46,980 オールインワン端末。Wi-Fi内蔵、レシート印刷可
Square Stand ¥36,980 iPad用POS端末。レジカウンター設置型
lightbulb
QRaku × Square のオンライン決済のみを使用する場合は、物理端末は不要です。お客様はスマホのブラウザ上でカード情報を入力して決済できます。

3. PayPay ダイレクト決済の導入手順
APIキー発行が必要 — 所要時間：約30分〜1営業日

1PayPay for Business に登録する

まず、PayPayの加盟店契約を行います。

link https://merchant.paypay.ne.jp/
  • 「加盟店お申し込みはこちら」をクリック
  • 法人情報 or 個人事業主情報を入力
  • 業種・業態として 「飲食業」 を選択
  • 銀行口座情報を登録
  • 審査完了メールが届くまで待つ（通常1〜3営業日）
info
PayPayの加盟店審査には1〜3営業日かかる場合があります。お急ぎの場合は、まず「現場決済」または「Square決済」でサービスを開始し、審査完了後にPayPayを追加することをおすすめします。
lightbulb
既にPayPayをご利用中の店舗様へ：
新たに加盟店申込（手順1）を行う必要はありません。ただし、API連携を利用したことがない場合は「手順2：PayPay for DevelopersでAPIキーを取得する」へ進んでください。
※既にAPIキーをお持ちの場合は、そのまま「手順4」へお進みください。

2PayPay for Developers でAPIキーを取得する

加盟店審査が完了したら、API連携用の認証情報を取得します。

link https://developer.paypay.ne.jp/
  1. PayPay for Developersにログイン
  2. 左メニューの 「ダッシュボード」 → 「APIキー管理」 を開く
  3. 以下の3つの情報をメモ・コピーしてください：
    • API Key — 英数字の文字列
    • API Secret — 英数字の文字列
    • 加盟店ID (Merchant ID) — 数字の文字列
warning
重要：API SecretはPayPay画面で一度しか表示されません。必ずメモ帳などに保存してから次のステップに進んでください。紛失した場合はAPIキーの再発行が必要です。

3本番 / サンドボックス環境の選択

PayPay for Developersでは、テスト用のサンドボックス環境と、実際に課金が発生する本番環境が分かれています。

環境用途注意点
サンドボックス テスト決済。実際のお金は動かない テスト用のPayPayアカウントが必要
本番 (Production) 実際のPayPay残高から引き落とす お客様の実際の残高から引き落とされます
lightbulb
初めて導入する場合は、まずサンドボックスのAPIキーでテストし、問題なければ本番のAPIキーに切り替えることを推奨します。

4QRaku 管理画面で認証情報を設定する

取得した3つの情報をQRaku管理画面に入力します：

  1. 管理画面 → 「決済設定」 ページを開く
  2. 決済方式から 「PayPay ダイレクト」 を選択
  3. 表示されるフォームに API Key、API Secret、加盟店ID を入力
  4. 「認証情報を保存」 ボタンをクリック
  5. 「✅ 認証情報設定済み」と表示されれば完了

4. QRaku 管理画面での設定方法
決済設定画面の操作ガイド

1決済設定画面へのアクセス方法

管理画面にログイン後、以下の方法で決済設定ページにアクセスできます：

  1. 管理画面上部のナビゲーションバーで 「決済」 タブをクリック
  2. または、URLバーに直接以下を入力：
link https://あなたのドメイン/{店舗ID}/admin/payment

2決済方式の切り替え

決済設定画面では、3つの決済カードが表示されます。使いたい決済方式のカードをクリックするだけで即座に切り替わります。

  • 現場決済 — QRakuの決済APIをバイパスし、自店舗で直接精算
  • Square 決済 — クレジットカード・電子マネー等のオンライン決済
  • PayPay ダイレクト — QRコードでPayPay残高から決済
info
決済方式を切り替えると、即座にお客様の決済フローに反映されます。営業時間外に切り替えることをおすすめします。

3連携の解除方法

  • Square： 「Square連携を解除」ボタンをクリック → 確認ダイアログで「OK」
  • PayPay： API Key、API Secret を空欄にして保存すれば認証情報がクリアされます
  • 解除後は、自動的に「現場決済」モードに戻ります

5. テスト・動作確認の方法
導入後、必ずテスト決済を行ってください

✓Square のテスト

  1. Square Developer ダッシュボードのサンドボックスモードでテスト用アカウントを作成
  2. QRakuの管理画面でSquare連携（サンドボックスモード）を完了
  3. テスト用テーブルQRコードをスキャンし、注文・決済を実施
  4. Square ダッシュボードの「トランザクション」で決済が記録されていることを確認
lightbulb
テスト用カード番号: 4532 0123 4567 8901（有効期限: 任意の未来日、CVV: 任意の3桁）

✓PayPay のテスト

  1. PayPay for DevelopersのサンドボックスAPIキーをQRakuに設定
  2. Sandbox用PayPayアプリ（またはWebテスト環境）を使用
  3. QRコード決済フローを最後まで実施
  4. PayPay Developer ダッシュボードの「テスト」タブで決済履歴を確認

✓本番切り替えチェックリスト

  • ✅ 返金（Refund）のフローも確認済み

6. よくある質問 (FAQ)
お困りの際はこちらをご参照ください

Q. Square連携ボタンを押してもエラーになります

A. 以下を確認してください：

  • Squareアカウントで日本の銀行口座が登録されているか
  • ブラウザのポップアップブロックが有効になっていないか
  • Squareのサービスステータスに障害が発生していないか

Q. PayPayの認証情報を忘れました

A. PayPay for Developersダッシュボードにログインし、「APIキー管理」からAPIキーを再発行してください。API Secretは再表示できないため、再発行が必要です。

Q. Square と PayPay を同時に使えますか？

A. 現在のQRakuでは、一度に1つの決済方式のみ有効にできます。ただし、「現場決済」モードでSquare端末とPayPay店頭QRコードを併用する使い方は可能です。

Q. 返金（Refund）はどうやりますか？

A.

  • Square： Squareダッシュボードの「トランザクション」から個別の決済を選択し、「返金」ボタンをクリック
  • PayPay： PayPay for Business管理画面から対象の取引を検索し、返金処理を実施

Q. 手数料を安くする方法はありますか？

A. PayPayは飲食業の場合、加盟店申込時に業種割引が適用される場合があります。また、Squareはキャンペーン期間中に手数料が割引されることがあるので、定期的に公式サイトをチェックしてください。
