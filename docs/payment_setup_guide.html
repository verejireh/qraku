1. はじめに

QRaku は 3つの決済方式 に対応しています。店舗様のご利用形態に合わせてお選びください。

方式特徴必要な契約
① 窓口支払い
(pay_at_counter)		食後に店舗レジで現金/カード決済。オンライン決済は使わない。不要
② Square 連携
(square_integrated)		カード・Apple Pay・Google Pay など。テイクアウト先払いも Square POS / プリンターと同期。Square アカウント
③ PayPay ダイレクト
(paypay_direct)		PayPay ユーザー向け。手数料が低く、日本国内のお客様に最適。PayPay for Business + PayPay for Developers
重要: テイクアウト（お持ち帰り）注文を受け付けるには ②または③ の契約が必要です。先払いが行われないと注文そのものが成立しない仕様になっています。

2. 事前に準備するもの

  1. 店舗名義の銀行口座（入金先）
  2. 代表者の本人確認書類（運転免許証など）
  3. 営業許可証（飲食店営業許可のコピー）
  4. メールアドレス（管理画面ログイン用）
  5. 法人の場合: 登記事項証明書

3. Square 決済 導入手順

3-1. Square アカウントを作る

  1. https://squareup.com/jp/ja にアクセスし、「無料でアカウント作成」を押す。
  2. メールアドレス・パスワード・店舗情報（業種: レストラン）を入力。
  3. 本人確認・事業情報審査を完了させる（通常 1〜3 営業日）。
  4. 審査通過後、銀行口座を登録して入金先を設定する。

3-2. ロケーション（店舗）を登録

  1. Square Dashboard → 「アカウントと設定」 → 「店舗情報・ロケーション」。
  2. 店舗名・住所・営業時間を登録し「有効（ACTIVE）」状態にする。
  3. 複数店舗を運営する場合でも、QRaku は自動的に「最初のアクティブな店舗」を選択します。
注意: 複数店舗を運営する場合、連携したい店舗だけを有効化しておいてください。後から個別選択する UI は現バージョンではありません。

3-3. QRaku 管理画面で連携ボタンを押す

  1. QRaku 管理画面にログイン → 左メニュー「決済設定」を開く。
  2. 決済方式で「② Square 連携」を選択。
  3. 「Squareと連携する」ボタンを押す。
  4. Square の承認画面にリダイレクトされるので、表示された内容を確認して「許可」を押す。
  5. 自動的に QRaku に戻り、「連携済み」と表示されれば完了。
連携で保存されるもの: access_token / refresh_token / merchant_id / location_id。自動取得されるため、オーナー様が手入力する項目はありません。

3-4. お客様側で使える決済手段

  • Visa / Mastercard / JCB / American Express / Diners
  • Apple Pay（iOS Safari）
  • Google Pay（Android Chrome）

3-5. Square 決済の手数料

方式手数料（2026年4月時点）
対面カード決済3.25%
オンライン・QRコード経由3.6%

※ 最新の手数料は Square 公式サイトでご確認ください。

4. PayPay ダイレクト 導入手順

4-1. PayPay for Business に加盟店登録

  1. https://paypay.ne.jp/business/ から「加盟店申込」を行う。
  2. 店舗情報・代表者情報・銀行口座を入力して提出。
  3. 審査（通常 7〜14 営業日）通過後、加盟店ID（Merchant ID） が発行される。

4-2. PayPay for Developers で API キーを取得

  1. https://developer.paypay.ne.jp/ にアクセスし、PayPay for Business と同じメールアドレスでログイン。
  2. 左メニュー「API キー管理」を開く。
  3. ステージング環境（テスト用）で「新しい API キーを発行」を押し、以下 2 つをメモ:
    • API Key（例: stg_xxxxxxxxxxxx）
    • API Secret（例: xxxxxxxxxxxxxxxx==）
  4. 動作テスト後、本番環境に切り替えて同じ手順で発行。
    • 本番キーには stg_ のプレフィックスが付きません。
秘密情報の取り扱い: API Secret は画面に一度しか表示されません。安全な場所（パスワード管理ツール等）に保存してください。

4-3. QRaku 管理画面に 3 つの値を入力

  1. 管理画面 → 「決済設定」 → 決済方式「③ PayPay ダイレクト」を選択。
  2. 以下 3 項目を入力:
    • API Key
    • API Secret
    • Merchant ID（加盟店ID）
  3. 「保存」ボタンを押す。「設定済み」の緑バッジが表示されれば完了。

4-4. PayPay の手数料

  • 2026年4月時点: 1.98%（Square より低い）
  • 入金サイクル: 月2回〜月6回（契約プランによる）

4-5. お客様側の決済フロー

  1. カートで「PayPay で決済する」を押す
  2. PayPay アプリまたは Web 画面に自動遷移、QR コードまたは金額を確認
  3. PayPay 残高・銀行口座・クレジットカードのいずれかで支払い
  4. QRaku に自動復帰 → 注文が確定 → 領収書ページ表示

5. 管理画面での店舗側設定

5-1. 基本情報（/admin）

  • 店舗名・営業時間・店舗テーマ（桜・コスモス等 8種類）を設定
  • ポイント発行率・受取設定・消費税表示設定

5-2. 決済設定（/admin/payment）

上記 3 章・4 章で説明した Square / PayPay の連携画面。

5-3. 運営管理（/admin/operation）

  • テーブルの追加・削除・編集
  • 注文確認方式（KDS / Square POS）の切り替え
  • 言語設定（日本語・英語・韓国語・中国語）
  • テイクアウト機能の ON/OFF トグル

5-4. スタッフ管理（/admin/staff-manage）

  • マスター PIN（店長用）の設定
  • スタッフの追加・削除・個人 PIN 設定

5-5. 表示設定（/admin）

厨房ページ・レジページ・スタッフページのうち、不要なページを非表示にできます（最低 1 つは ON が必須）。

6. テイクアウト機能 有効化手順

STEP 1: 「決済設定」で Square または PayPay のどちらかを連携する
（テイクアウトには先払いが必須のため、オンライン決済が未設定だと機能しません）
STEP 2: 「運営管理」ページで「テイクアウト」トグルを ON にする
STEP 3: 「メニュー管理」で、テイクアウト可能なメニュー 1 品ずつに「Takeout OK」トグルを ON にする
STEP 4: 「QR Builder」ページで「テイクアウト」タブに切り替え、QR コードを 1 枚印刷して店内の目立つ場所（レジ横、入口など）に掲示する

6-1. テーブル客が同時にテイクアウトを注文する場合

  • テーブル QR からアクセスしたお客様には、メニューカテゴリの末尾に「🥡 テイクアウト」カテゴリが自動表示される
  • このカテゴリで注文したアイテムは「🥡 포장」バッジ付きで厨房に表示される
  • お会計時、食事代と一緒にレジで一括後払い

6-2. テイクアウト専用 QR を読んだお客様の流れ

  1. 店頭 QR をスキャン → テイクアウト可能メニューだけが表示される
  2. カートに追加 → 「注文する」ボタン
  3. 受取予定時刻を選択
  4. Square / PayPay で先払い
  5. 決済完了後、注文が厨房に送信され、4 桁の受取コードが発行される
  6. 完成したら Kitchen → Register で「受取準備完了」 → スタッフが受取コードを確認して商品を渡す

7. 動作テスト（サンドボックス / 本番）

7-1. Square サンドボックステスト

本番運用を始める前に必ずサンドボックス（テスト環境）で動作確認してください。

  1. システム管理者に SQUARE_ENVIRONMENT=sandbox でサーバー起動してもらう
  2. Square Developer Dashboard でサンドボックス用アカウントを作成
  3. テスト用カード番号で決済テスト:
    • 成功: 4111 1111 1111 1111 / CVV 111 / 有効期限: 将来日付
    • 失敗: 4000 0000 0000 0002
  4. 厨房画面に注文が表示されることを確認
  5. レシートページが正しく生成されることを確認

7-2. PayPay ステージングテスト

  1. API キーが stg_ で始まっているかを確認（QRaku が自動でステージング API に振り分け）
  2. PayPay for Developers から配布される「テスト用 PayPay アプリ」をスマホにインストール
  3. テスト決済を実行し、ステータスが COMPLETED になることを確認

7-3. 本番環境への切り替え

  1. Square: 管理画面で一度 OAuth 解除 → 本番環境に切り替え → 再連携
  2. PayPay: 本番用 API Key/Secret に書き換えて保存
  3. 実際に 1 件少額注文（100〜500円）を入れて入金を確認

8. トラブルシューティング

症状原因・対処
「テイクアウト」ボタンがグレーアウトして押せない運営管理でテイクアウトが OFF です。ONにしてください。
決済画面で「カード入力欄が出てこない」Square 連携が切れています。決済設定で再連携してください。
PayPay 決済後、注文が作成されないPayPay 完了ページへのリダイレクト URL が誤っています。システム管理者に FRONTEND_BASE_URL の設定確認を依頼してください。
Kitchen 画面に注文が出ないテイクアウト注文は決済完了後のみ表示されます。未決済は表示されません。
「Authentication required」エラー管理画面のセッションが切れています。再ログインしてください。
Square トークン期限切れ（約 90日）決済設定で OAuth を解除し、再度「Square と連携する」を押してください。

9. よくある質問

Q1. 複数の決済方式を同時に使えますか？

1 店舗 1 方式までです。Square 連携中は PayPay ダイレクトと併用できません。テイクアウト用に PayPay、イートインはレジ後払い、という運用は可能です。

Q2. 手数料はいつ・どう引かれますか？

Square / PayPay のそれぞれの入金サイクルで、売上から手数料を差し引いた金額が銀行口座に振り込まれます。QRaku 側では手数料を取りません。

Q3. 領収書は発行できますか？

お客様は注文後のレシートページ（/receipt/:orderId）をスマホで保存可能です。紙の領収書が必要な場合はレジでスタッフに申し出てください。

Q4. 返金はどうしますか？

Square: Square Dashboard → 注文履歴から返金処理。PayPay: PayPay for Business の管理画面から返金処理（API 経由も可能、システム管理者にご相談ください）。

Q5. QR コードを紛失・破損したら？

「QR Builder」ページから同じ QR を何度でも再印刷できます。セキュリティ上の問題はありません（QR は店舗 ID のみを含む公開情報）。

Q6. マスター PIN を忘れたら？

管理画面（/admin/staff-manage）にログインできれば再設定可能です。管理画面ログイン自体を忘れた場合はシステム管理者にパスワードリセットを依頼してください。
